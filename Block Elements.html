Block Elements

Key differentiator

A block element, if no width is set, will expand naturally to fill its parent container. Also, by default, it will be placed below previous elements in the markup.

TLDR:

  • <address>
  • <blockquote>
  • <ol>
  • <video>
  • <div>

<address>

This element indicates that the enclosed HTML provides contact information for a person or people, or for an organization.

This is another HTML tag providing useful semantic meaning to help visually impaired identify addresses.

<address>

  Bitwise Industries<br>

  700 Van Ness Ave.<br>

  Fresno, CA 93721

</address>

Do This:

  • From the Terminal, use the touch command to create a new block.html file.
  • Using the Emmet ! shortcut, add in the page scaffolding
  • Inside the <body> tag, add an <address> tag and put in your address.

Note: There is a small element, <br>, in here that breaks a line of text to a new line. It's called a line break.

<blockquote>

This element indicates that the enclosed text is an extended quotation.

If you need to represent a client testimonial or similar quote, the <blockquote> is your best option.

<blockquote cite="http://www.moviequotedb.com/movies/jurassic-park/quote_12179.html">

  <p>God creates dinosaurs. God destroys dinosaurs. God creates man. Man destroys God. Man creates dinosaurs.</p>

</blockquote>

Surrounding a <p> tag with the <blockquote> changes the meaning of the paragraph to an actual citeable quote. Notice the cite attribute providing the link to the quote location.

Do This:

  • After the <address> tag in your block.html file, create a short paragraph tag with some quoteable text.
  • Now wrap that <p> tag with the <blockquote> and adjust your indentions appropriately.
  • Try to find a source for your quote online and use the cite attribute.

<ol>

This element represents an ordered list of items, typically rendered as a numbered list.

Virtually identical to the <ul> except creating a numbered list instead of bulleted.

The list of items are marked up with <li> tags creating a parent/child relationship. If you want to use an <li>, it must be nested inside a <ol>. If you want to use the <ol>, the immediate child element must be an <li>.

<ul>

  <li>Pepsi</li>

  <li>Mt. Dew</li>

  <li>Root Beer</li>

</ul>

Do This:

  • Using your block.html page, make an ordered list of your top three favorite ice cream flavors.

<video>

Use this element to embed video content in a document.

Ya know... 'cuz video is cool.

<video src="https://storage.googleapis.com/coverr-main/mp4/Play_Date.mp4" width="350px" controls></video>

There are more element attributes than any other we've seen up to this point. The src attribute loads the URL to the webpage. The width and it's value declare a width for the element and controls makes basic controls of the video interactive.

Do This:

  • After the your <ol> element, add a <video> element
  • Choose a video snippet from https://coverr.co/
  • View the video in a new tab (we'll walk through this process)
  • "Hotlink" one of thest videos into your webpage

Note: Hotlinking images is bad. After today, don't do it. Last time, I swear.

<div>

The division element is a generic container for flow content. It has no effect on the content and does not inherently represent anything.

Basically, if you need specific styling applied to a group of elements and want to prevent incorrect defining of content, use a <div>. You'll use a lot of them.

<div>

  <img src="https://unsplash.it/250/250" alt="placeholder image">

  <p>landitiis obcaecati minus, corporis laudantium iste fuga, cumque nobis saepe quos a voluptatem fugiat iure soluta eaque impedit.</p>

  <a href="anoter_page.html">Learn More</a>

</div>

The above example may best be represented as an abbreviated biographical summary linking to another page providing additional and more detailed information.
