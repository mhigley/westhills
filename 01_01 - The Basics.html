git init

Creates a new repository from a folder

Do This:

  • Open the Terminal
  • At the prompt, lets cd (change directory) to the Desktop
  • cd ~/Desktop

Note: You'll see me use the ~ (tilde: pronounced til-duh) a lot. From wherever you are, starting with the tilde always brings you back to the home folder.
Another shortcut back to your home folder is just the cd command. Try it. Bury yourself deep into the system then as a standalone command, type cd. Shibby.

  • Now type mkdir my-project

Note: mkdir is exactly what is sounds like when you sound it out. "Make Dir(ectory)"

  • Now lets cd into the new folder we just made.
  • cd my-project
  • Now that we are inside the folder we want to track, lets initialize git with git init

Let's create a couple files inside this project folder

  • Type touch index.html. This will create a new blank HTML document.
  • Try typing code . (This should open the project folder inside of VSCode)
  • Double click the index.html in the sidebar to edit.
  • On line 1, type ! then press tab. Shazam! Insta-code!
  • Between the open <body> tag and the closing </body> tag, let's add some markup
<h1>Headline for the Index page</h1>

<p>Between the open and closing paragraph tags, you can type whatever you like</p>
  • Be sure to save your work with ctl + s (win) or âŒ˜ + s (mac)
  • Go back to the Terminal and try ls. This lists the names of the files in your current working directory. Try adding on some additional options like -a, or -al.

Let's duplicate this file now from the Terminal

  • cp index.html about.html
  • Try ls with a variation of options again.

git status

Git status shows you the current state of all edited or staged files.

Do This:

  • Type git status
    Try it out. Notice that it writes out all the files and lists them as Untracked. As of right now, git knows nothing about these two files.

git add

Git add moves files from being untracked into a staging environment.

There are two different ways of adding files to staging. First we'll add just a single file.

Do This:

  • git add index.html
  • Do another git status, what has changed?

Note: Adding files is kinda like saving your work, but nothing is set in stone. You can keep making edits and changes and add more work.

  • Go back into VSCode and make a change to your index.html file.
  • Then do another git status

Git knows about the file and recognizes that changes have been made but have not been added yet.
Let's try another way of adding files.

  • git add . (That is a space after add and then a period)

This adds all new and modified files to be staged.

  • git status

git commit

Git commit creates a more permanent record of files and folders created or modified.

This one is a doozey. You're accustomed to git. Next, we want a more permanent record of your saved changes with commit. After that is an option flag to add a message with -m (not optional). Last is the actual message with "Your message here". Let's commit this mug.

Do This:

  • git commit -m "Adding first pages"

Note: You may be propted after first running this command to save some configuration data. Run both of the lines of code with your information replacing the sample text
git config --global user.name "Your Github Name"
git config --global user.email "Your Github Email Address"

  • I believe if you had to supply the above information, you'll have to run the commit command again.
  • Try out the clear command to clean up a long and messy Terminal window.
  • Run git status again. Notice it's message: "On branch master. Nothing to commit." (We'll discuss branches soon)

git log

Imagine if you were to print out a list of all the changes you made. It would be "permanent". That is what git log does. It prints out all the committed changes.

Do This:

  • git log

Notice the information it provides you.

First, a unique tracking number identifying your commit. Second it shows who and when the commit was made by. Lastly, it prints out the first line of your commit message.
