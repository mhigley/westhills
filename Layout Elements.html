Layout Elements

New HTML (HTML5) offers new semantic elements to define different parts of a web page layout.

The page layout is the most complicated part of web development. Understanding how to create semantic containers to guide the flow

TLDR:

  • <nav>
  • <header>
  • <article>
  • <footer>

Note: All layout tags share block-level attributes!

<nav>

This element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.

The two highlightes words in this description are key aspects to the proper usage of these layout elements. They create a "meaning" for, not a thing, but a "group" of things.

<nav>

  <ul>

    <li><a href="http://www.imdb.com/title/tt0107290/?ref_=nv_sr_1">Jurassic Park</a></li>

    <li><a href="http://www.imdb.com/title/tt0119567/?ref_=nv_sr_2">The Lost World: Jurassic Park</a></li>

    <li><a href="http://www.imdb.com/title/tt0163025/?ref_=nv_sr_3">Jurassic Park III</a></li>

  </ul>

</nav>

Notice that this is the most complicated nesting of HTML elements that we've come across. Navigations are one of the more intricate parts of a website, especially if we get into multiple sub-levels of navigation.

If you break my example down to it's smallest components, I originally started with three anchor tags. Grouped together, they create a list of navigational items so each <a> tag needs to be indivicually wrapped in an <li> tag. Because list items cannot stand alone, I needed to contain all of the items inside of a <ul> tag. An <ol> isn't appropriate here, because there is no greater importance of one over another. Lastly, as it is, this list is nothing more than a list. I need the browser to know this is actual navigation to an outside destination, so I wrapped the entire <ul> inside of a <nav> tag. Done!

Do This:

  • From the Terminal, use the touch command to create a new layout.html file.
  • Using the Emmet ! shortcut, add in the page scaffolding
  • Inside the <body> tag, start by adding three to five <a> tags with link text and a destination URL for the href
  • Next, make each of the links into a proper unordered list
  • Lastly, wrap that list with the <nav> tags to declare is navigation

<header>

This element represents introductory content, typically a group of introductory or navigational aids. It may contain some heading elements but also other elements like a logo, a search form, an author name, and so on.

Another element that provides much greater meaning to the content that is laid out within it. It can contain the site logo, the entire site-wide navigation, links to social media, and way more!

<header>

  <img src="img/site_logo.png" alt="Sample Website">

  <nav>

    <ul>

      <li><a href="#">Navlink 1</a></li>

      <li><a href="#">Navlink 1</a></li>

      <li><a href="#">Navlink 1</a></li>

    </ul>

  </nav>

  <address>

    500 Lane Ave.<br>

    Somecity, CA 90001

  </address>

</header>

Note: Don't mix up <head> and <header>. They are two very different elements. The <head> tag can only be used before the <body> tag, and the <header> tag can only be use inside the <body> tag.

Do This:

  • Continuing work on the layout.html page, add additional information you feel would be found atop a website.
  • Think social media, maybe a login field, a company logo, etc...
  • Being careful to enclose everything that you just wrote, wrap it all with a <header> tag defining that content.

<article>

This element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable (e.g., in syndication). Examples include: a forum post, a magazine or newspaper article, or a blog entry.

This tag is very specific as the definition implies. Imagine that if you were to delete every aspect of your website except the composition, or blog post, could it stand alone? Does the reader get all relavent headlines, subheadings, visuals, author information and content needed to understand the story?

<article>

  <h1>My Article Headline</h1>

  <h2>The Understanding of Complex Design Patterns in UI/UX</h2>

  <p>Lorem ipsum dolor sit, amet consectetur adipisicing elit. Quaerat commodi est in, cum ipsum molestias provident exercitationem aliquid facilis deserunt ad quos porro, et recusandae? Omnis dolorem quidem aliquam quae!</p>

  <p>Amet fuga enim recusandae maiores itaque voluptatum molestiae culpa alias, eaque, numquam tenetur est commodi sapiente voluptatem consequatur delectus eius corporis similique mollitia? Ipsam, perferendis.</p>

  <p>

    Author: Matthew Higley<br>

    <small>Mar 2017</small>

  </p>

</article>

Do This:

  • In your layout.html page, create some content.
  • Think of this content in the form of telling a complete story.
  • Include at least a headline, some content and one image.
  • Be sure to wrap everything together as a stand-alone syndicatable story

<footer>

This element represents a footer for its nearest sectioning content or sectioning root element. A footer typically contains information about the author of the section, copyright data or links to related documents.

More often you'll find the <footer> as a near last item of a website. Links for additional information or reference, social media, contact information, and copyright text are all things you can find nested inside a <footer>.

<footer>

  <p>Copyright 2018. All rights reserved.</p>

  <ul>

    <li><a href="https://facebook.com">facebook</a></li>

    <li><a href="https://twitter.com">twitter</a></li>

    <li><a href="https://instagram.com">instagram</a></li>

  </ul>

</footer>

Do This:

  • At the bottom of your layout.html page, create some generic copyright information.
  • Add in another block of contact information.
  • Wrap everything together that you have determined as footer content inside of the <footer> tag.
