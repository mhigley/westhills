Inline Elements

Key differentiator

Inline elements do not clear previous content to drop to the next line like block elements.

TLDR:

  • <em>
  • <strong>
  • <input>
  • <button>
  • <span>

<em>

This element marks text that has stress emphasis.

Inline elements such as the <em> tag flows along with text content and should be used as such. The element is often used to indicate an implicit or explicit contrast.

<p>T. Rex doesn't want to be <em>fed</em>, he wants to <em>hunt</em>. Can't just... suppress 65 million years of gut instinct.</p>

Do This:

  • From the Terminal, use the touch command to create a new inline.html file.
  • Using the Emmet ! shortcut, add in the page scaffolding
  • Inside the <body> tag, create a <p> tag and 100 words of lorem ipsum.
  • Somewhere in your paragraph, open up an <em> tag and a dozen or so words later, close your </em> tag.

<strong>

This element indicates that its contents have strong importance, seriousness, or urgency.

This could be used to merely try to point out that some words are of greater importance compared to nearby content.

<p>Mr. Hammond, after careful consideration, I've decided <strong>not</strong> to endorse your park.</p>

Do This:

  • Choose a few word of your lorem ipsum and wrap them in an open <strong> and close with a closing </strong> tag.

Danger! Besure you don't cross your HTML tags! Yes, you may nest them, but don't close a pair in the middle of another pair.

<p>This is an example of how <strong><em>not</strong></em> to use inline elements</p>

<input>

This element is used to create interactive controls for web-based forms in order to accept data from the user.

The <input> is a key feature in gathering information from your user. In a static website, you'll have no way of capturing that data, but we'll deal with that later.

<input type="text">

Notice the type attribute inside the <input> tag. There are more than 20 different types and the types vary in functionality. The basic type="text" captures letters, numbers and potentially special characters.

Do This:

  • After your paragraph, create an <input> tag that captures user text.
  • When you finish, figure out how to have "helper text" visible in the box guiding the user as to proper input

<button>

The <button> tag defines a clickable button.

This element requires both an open and closing tag. The text you put between will be the button text.

<button type="button">Click Here</button>

Notice I've included a type for a button. This element has three potientially different defaults depending the users browser.

Do This:

  • On the line after your <input>tag, create a <button> with text instructing the user.

Note: Typically, the <button> element is for interactivity. There is a different element to submit the capture of user data.

<span>

This is a generic inline container for phrasing content, which does not inherently represent anything.

If you need specific styling applied to a string of text preventing any defining of content, use a <span>.

<p>There is a section of content beginning here <span>that contains <strong>important</strong> or <em>accentuated</em> words</span> I want to target</p>

There are the two words in the example that have appropriate HTML tags applied defining them. However, a portion of the sentence can be wrapped with a <span> tag that enables the ability to be highlighted without affecting their meaning.
